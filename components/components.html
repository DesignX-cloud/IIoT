針對不同行為，用不同顏色代表之

.btn-info

call to action，實心按鈕，多數為CRUD的C增加（Create）之行為 ， 所以同時是進行步驟順序引導使用顏色，進入功能頁面/區塊，第一順位行為，不具引導順序button event不使用實心按鈕，使用 btn-outline-*。

.btn-outline-success

多數為CRUD之U改正（Update，意為「更新」）。

live preview
< button>class=" btn-outline-success text-base btn btn-md">... < /button>
.btn-outline-info

多數為CRUD之U改正（Update，「更新」），但非功能頁裡主要之行為，為執行主要功能後才會產生之附屬行為。

live preview
< button>class=" btn-outline-primary text-base btn btn-md">... < /button>
.btn-outline-primary

多數為CRUD之R（查詢（Read，意為「讀取」））。

live preview
<buttonclass=" btn-outline-primary text-base btn btn-md rounded-circle">... < /button>
.btn-outline-danger

多數為CRUD之D（刪除Delete）。

<button>class=" btn-outline-danger text-base btn btn-md">... < / button>
.btn-outline-warning

在執行CRUD之D（刪除Delete）或部分（Update，「更新」）前的提醒警告。
